Domande frequenti

Vale la pena migrare un progetto JavaScript a TypeScript?

Quasi sempre sì se il progetto sopravvive più di 6 mesi. Il refactoring sicuro, la documentazione 'a codice' e l'IDE migliore ripagano l'investimento iniziale in 1-2 mesi.

'strict: true' o partire morbidi?

Strict mode è il default che consiglio per nuovi progetti. Per migrazioni progressive si parte morbidi e si aumenta gradualmente: noImplicitAny → strictNullChecks → strict completo.

Type-safety end-to-end tra backend e frontend, è possibile?

Sì, con tRPC, TypeScript shared packages in monorepo, o OpenAPI con codegen. La scelta dipende dall'architettura: tRPC se Node fullstack, codegen OpenAPI se backend è in altri linguaggi.

TypeScript rallenta lo sviluppo?

Nei primi 1-2 mesi sì del 10-20%. Dopo diventa più veloce di JavaScript grazie a meno bug, refactoring più audaci, IDE più utile. ROI fortemente positivo.

Lavorate solo con TypeScript o anche JavaScript classico?

Default TypeScript per tutto. Su progetti esistenti in JavaScript posso lavorare in entrambi i modi e pianificare migrazione progressive.
